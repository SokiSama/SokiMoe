其实这两天也想折腾 Mix Space + Shiro 来搭建一个自己的小站，因为确实好看。但是它是需要 Docker 加上涉及后端，想了想还是继续写纯前端吧。

这个博客其实是 V2EX 一个老哥开源的，我只是在他的基础上进行了一些修改。

也是我目前发现最接近 Shiro 主题的了。在修改主题中，部分的灵感源于各大博客主的设计。

我为什么抛弃了 Astro + Mizuki 主题，又放弃了 Hexo呢。原因有下

1. 极速的加载体验 (性能优势)
    Next.js 最核心的优势在于它的渲染模式。

    SSG（静态生成）： 博客文章通常不会秒级更新。Next.js 会在构建时将你的 Markdown 或 CMS 内容直接生成 HTML。

    秒开体验： 读者访问时，浏览器直接获取静态文件，配合 Vercel 等 CDN 加速，几乎是点击即开。

2. 极其自由的视觉定制 (Tailwind CSS)
作为有些简约审美的我，Tailwind 是很好的选择。

    无需离开 HTML： 不需要在 CSS 文件和 JS 文件之间反复横跳。所有的间距、颜色、排版都通过类名（Utility Classes）直接搞定。

    原子化设计： 就像拼乐高，你可以轻松做出非常有设计感的排版，而不用担心 CSS 全局污染。

    暗黑模式原生支持： 只需要加一个 dark: 前缀，就能完美适配深夜码字读者的眼睛。

3. 开发效率与生态
    全栈能力： 即使它只是个博客，但如果你想加个“留言板”或者“点赞按钮”，Next.js 自带的 API Routes 可以让你直接写后端逻辑，无需额外服务器。

    图片优化： Next.js 的 next/image 组件会自动根据屏幕大小缩放图片并转换为 WebP 格式，这对于放了很多截图的博客非常有用，能省下大量流量。

4. 零成本或低成本维护
    部署简单： 关联 GitHub 仓库到 Vercel 平台上，每次你推送代码或更新文章，博客就会自动重新构建发布。

    免费额度： 对于个人博客的流量，Vercel 的免费套餐完全够用。


总之，这次我推翻了所有的框架与主题，纯手搓重构这个博客。虽然它部分功能和审美不如其他已上线的主题。但是至少我可以根据自己的需求增加与删除功能。
